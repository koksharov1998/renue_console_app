Есть csv файл с данными по аэропортам. Необходимо сделать консольное приложение. В настройках приложения (в свойствах) задается номер колонки из файла с данными, поиск по которой будет происходить. Программа в консоли ждет ввода строки. Пользователь вводит строку и получает список отфильтрованных значений аэропортов по этой Приложение на спринге, лучше если будет spring boot, jdk 11, maven.
Программа должна после вывода аэропорта вывести количество найденных строк и время в миллисекундах, потраченное на поиск.
колонки по введенному префиксу в лексикографическом порядке.

Поиск должен быть максимально быстрый, памяти должно использоваться по минимуму. Принципы ООП соблюдаться.

Критерии проверки: 
1. для работы приложения на приложенном aiports.csv требуется не более 9 мб хипа для работы(-Xmx9m). 
2. скорость поиска должна быть высокой и не зависеть от общего числа строк.
3. ошибочные и краевые ситуации должны быть обработаны

Пример: 
Пользователь запускает программу, вызывая java -jar <имя файла сборки> <необязательный номер колонки(по умолчанию из настроек берется)>.
Программа выводит запускается и пишет "Введите строку:"
Например, в настройках (application.yml) указана 2-ая колонка для индексирования. Пользователь вводит "Bo" в консоль, нажимает Enter, получает результаты:
Bou Chekif Airport ... (остальная инфа по аэропорту)
Bou Sfer Airport ... (остальная инфа по аэропорту)

Количество найденных строк: 2
Время, затраченное на поиск: 2 мс.
